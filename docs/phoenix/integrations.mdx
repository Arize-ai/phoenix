## Integration Types

Phoenix has a wide range of integrations. Generally these fall into a few categories:

1. **Tracing integrations** - where Phoenix will capture traces of applications built using a specific library.

   a. *E.g.* [*OpenAI*](/phoenix/integrations/llm-providers/openai)*,* [*LangChain*](/phoenix/integrations/python/langchain)*,* [*Vercel AI SDK*](/phoenix/integrations/typescript/vercel/vercel-ai-sdk-tracing-js)*,* [*Amazon Bedrock*](/phoenix/integrations/llm-providers/amazon-bedrock)*,* [*Hugging Face smolagents*](/phoenix/integrations/python/hugging-face-smolagents)

2. **Eval Model integrations** - where Phoenix's eval Python package will make calls to a specific model.

   b. *E.g.* [*OpenAI*](/phoenix/integrations/llm-providers/openai)*,* [*Anthropic*](/phoenix/integrations/llm-providers/anthropic)*,* [*Google VertexAI*](/phoenix/integrations/llm-providers/vertexai)*,* [*Mistral*](/phoenix/integrations/llm-providers/mistralai)

3. **Eval Library integrations** - where Phoenix traces can be evaluated using an outside eval library, instead of Phoenix's eval library, and visualized in Phoenix.

   c. *E.g.* [*Ragas*](/phoenix/integrations/evaluation-integrations/ragas)*,* [*Cleanlab*](/phoenix/integrations/evaluation-integrations/cleanlab)

Each partner listing in this section contains **integration docs** and **relevant tutorials.**
