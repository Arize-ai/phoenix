Evaluating AI agents requires fundamentally different approaches than evaluating standalone LLM calls. Agents operate across multiple steps, use tools, maintain state, and make decisions that compound — meaning failures are more varied, harder to detect, and more consequential than in single-turn systems.

This guide provides a practical, research-backed framework for agent evaluation — from understanding failure modes to building layered evaluation systems that catch issues at every level.

## Who This Guide Is For

This guide is designed for teams building, deploying, or procuring AI agent systems — including ML engineers designing evaluation pipelines, technical program managers defining acceptance criteria, and engineering leaders making build-vs-buy decisions about agent infrastructure.

The framework is tool-agnostic: while examples reference Arize's evaluation platform, the methodology applies to any agent evaluation workflow.
