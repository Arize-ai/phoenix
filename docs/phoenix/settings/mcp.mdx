Phoenix ships a remote [MCP](https://modelcontextprotocol.io/) server built into the Phoenix server. The **Settings → MCP** tab surfaces everything you need to connect a client without leaving the product.

## What the tab shows

- **Server status** — whether the MCP server (`PHOENIX_ENABLE_MCP_SERVER`) and code mode (`PHOENIX_ENABLE_MCP_CODE_MODE`) are enabled on this deployment. If the server is disabled, the tab explains how to turn it on.
- **MCP Server URL** — your Phoenix base URL plus `/mcp`, with a copy button. Point any MCP-compatible client at this URL.
- **Quick setup** — the [`px setup mcp`](/docs/phoenix/sdk-api-reference/typescript/arizeai-phoenix-cli) command, which infers your endpoint and writes your coding agent's MCP config for you.
- **Connect a client** — per-client, copy-paste setup instructions (Claude Code, Claude Desktop, Cursor, VS Code, OpenCode, Codex, Antigravity, and others). The copy adapts to whether authentication is enabled: with auth, each client walks through its OAuth login; without auth, the URL alone is a complete config.

## Learn more

The [Remote MCP Server](/docs/phoenix/integrations/remote-mcp) integration page covers the same setup in depth — the endpoint, OAuth vs. API-key authentication, code mode, the configuration environment variables, and manual per-client configuration. See that page for the full reference.
