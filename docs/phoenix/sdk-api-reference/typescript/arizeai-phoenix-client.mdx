TypeScript client for the [Arize Phoenix](https://github.com/Arize-ai/phoenix) API. Use it to manage prompts, datasets, experiments, spans, sessions, and raw REST calls against a Phoenix server.

## Installation

```bash
npm install @arizeai/phoenix-client
```

<Note>
`@arizeai/phoenix-client` ships local docs, mirrored examples, and source code inside `node_modules/@arizeai/phoenix-client/`.

- `docs/overview.mdx`
- `docs/examples/`
- `src/`

Browse the website version under [@arizeai/phoenix-client Docs](/docs/phoenix/sdk-api-reference/typescript/packages/phoenix-client/overview).
</Note>

## Local Install Pattern

After install, a coding agent can inspect:

```text
node_modules/@arizeai/phoenix-client/docs/
node_modules/@arizeai/phoenix-client/docs/examples/
node_modules/@arizeai/phoenix-client/src/
```

That gives the agent version-matched prose, runnable example code, and the exact implementation and type definitions that shipped with your project.

## Recommended Starting Files

- `docs/configuration.mdx` for `PHOENIX_HOST`, `PHOENIX_API_KEY`, and client creation
- `docs/examples/list_datasets.ts` for raw typed REST access
- `docs/examples/create_prompt_openai.ts` and `docs/examples/apply_prompt_vercel.ts` for prompt workflows
- `docs/examples/run_experiment.ts` and `docs/examples/run_experiment_with_evals.ts` for experiments
- `docs/examples/get_spans.ts`, `docs/examples/get_traces.ts`, and `docs/examples/get_session_turns.ts` for retrieval flows
