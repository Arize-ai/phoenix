A simple wrapper around OpenTelemetry for use with [Arize Phoenix](https://github.com/Arize-ai/phoenix). This package simplifies the setup of OpenTelemetry tracing for Node.js applications, providing an easy way to send traces to Phoenix for observability and analysis.

> **Note**: This package is still under active development and is subject to change.

## Installation

```bash
npm install @arizeai/phoenix-otel
```

<Note>
`@arizeai/phoenix-otel` ships local docs, mirrored examples, and source code inside `node_modules/@arizeai/phoenix-otel/`.

- `docs/overview.mdx`
- `docs/examples/`
- `src/`

Browse the website version under [@arizeai/phoenix-otel Docs](/docs/phoenix/sdk-api-reference/typescript/packages/phoenix-otel/overview).
</Note>

## Local Install Pattern

After install, a coding agent can inspect:

```text
node_modules/@arizeai/phoenix-otel/docs/
node_modules/@arizeai/phoenix-otel/docs/examples/
node_modules/@arizeai/phoenix-otel/src/
```

## Recommended Starting Files

- `docs/examples/register_example.ts` for the minimal setup path
- `docs/examples/manual_spans_example.ts` for custom spans
- `docs/examples/global_provider_lifecycle_example.ts` for `global: false` plus manual attach/detach
- `docs/examples/instrumentation_example.ts` for CommonJS auto-instrumentation
