`@arizeai/phoenix-evals` provides evaluator building blocks for TypeScript workflows. It includes LLM-based evaluators, code-based evaluators, prompt templating helpers, and compatibility points for Phoenix experiments.

## Install

```bash
npm install @arizeai/phoenix-evals
```

## Docs And Source In `node_modules`

After install, a coding agent can inspect the installed package directly:

```text
node_modules/@arizeai/phoenix-evals/docs/
node_modules/@arizeai/phoenix-evals/src/
```

## Best Starting Pages

- `docs/installation.mdx` for provider requirements and install combinations
- `docs/create-evaluator.mdx` and `docs/code-evaluators.mdx` for deterministic evaluator flows
- `docs/llm-evaluators.mdx`, `docs/built-in-metrics.mdx`, and `docs/classification.mdx` for model-backed evaluation
- `docs/templates.mdx` and `docs/phoenix-integration.mdx` for prompt helpers and experiment wiring

## Module Map

- Root export: all supported helper modules
- `@arizeai/phoenix-evals/llm`: classification and built-in LLM evaluators
- `@arizeai/phoenix-evals/template`: prompt templating helpers
- `@arizeai/phoenix-evals/helpers`: evaluator construction helpers
- `@arizeai/phoenix-evals/types`: shared evaluator types
