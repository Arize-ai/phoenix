`@arizeai/phoenix-client` is the typed TypeScript client for Phoenix platform APIs. It ships a small root REST client plus focused module entrypoints for prompts, datasets, experiments, spans, sessions, and traces.

## Install

```bash
npm install @arizeai/phoenix-client
```

## Docs, Examples, And Source In `node_modules`

After install, a coding agent can inspect the installed package directly:

```text
node_modules/@arizeai/phoenix-client/docs/
node_modules/@arizeai/phoenix-client/docs/examples/
node_modules/@arizeai/phoenix-client/src/
```

That gives you version-matched docs, runnable examples, and the exact implementation that shipped with the current project. No network access is required.

## Best Starting Files

- `docs/configuration.mdx` for client setup and environment variables
- `docs/examples/list_datasets.ts` for raw typed REST access
- `docs/examples/create_prompt_openai.ts` and `docs/examples/apply_prompt_vercel.ts` for prompt workflows
- `docs/examples/run_experiment.ts` and `docs/examples/run_experiment_with_evals.ts` for experiments
- `docs/examples/get_spans.ts`, `docs/examples/get_traces.ts`, and `docs/examples/get_session_turns.ts` for retrieval workflows

## Module Map

- Root export: `createClient`, generated OpenAPI types, and config helpers
- `@arizeai/phoenix-client/prompts`: prompt CRUD plus `toSDK`
- `@arizeai/phoenix-client/datasets`: dataset creation and retrieval helpers
- `@arizeai/phoenix-client/experiments`: experiment execution and lifecycle helpers
- `@arizeai/phoenix-client/spans`: span search, notes, and annotations
- `@arizeai/phoenix-client/sessions`: session listing, retrieval, and annotations
- `@arizeai/phoenix-client/traces`: project trace retrieval
