## Overview

Phoenix CLI 0.7.0 adds **sessions** support — multi-turn conversation tracking for LLM applications. New REST API endpoints and CLI commands let you list sessions, inspect conversation flows, and review session annotations directly from your terminal.

```bash
# List recent sessions
px sessions --limit 5

# View a session's conversation flow
px session <session-id>

# Include annotations (scores, labels)
px session <session-id> --include-annotations

# Export to file
px session <session-id> --file session-data.json
```

## Sessions REST API

Three new endpoints join the existing `POST /v1/session_annotations`:

| Method | Endpoint | Description |
|--------|----------|-------------|
| `GET` | `/v1/projects/{project_identifier}/sessions` | List sessions for a project |
| `GET` | `/v1/sessions/{session_identifier}` | Get a session by ID |
| `GET` | `/v1/projects/{project_identifier}/session_annotations` | List session annotations |

## CLI Commands

### `px sessions` — List Sessions

```bash
px sessions [options]
```

| Flag | Default | Description |
|------|---------|-------------|
| `--project <name>` | — | Project name or ID |
| `-n, --limit <number>` | `10` | Maximum sessions to return |
| `--order <order>` | `desc` | Sort order: `asc` or `desc` |
| `--format <format>` | `pretty` | Output format: `pretty`, `json`, or `raw` |
| `--no-progress` | — | Disable progress indicators |

### `px session` — View a Session

```bash
px session <session-id> [options]
```

| Flag | Default | Description |
|------|---------|-------------|
| `--project <name>` | — | Project name or ID |
| `--include-annotations` | — | Include session annotations |
| `--file <path>` | — | Save to file (forces JSON output) |
| `--format <format>` | `pretty` | Output format: `pretty`, `json`, or `raw` |
| `--no-progress` | — | Disable progress indicators |

## Built for AI Coding Agents

Sessions commands are designed to pipe context into AI coding assistants like Claude Code and Cursor. Fetch multi-turn conversation data and feed it directly into your agent's context:

```bash
# Get the last 5 sessions as JSON for an AI assistant
px sessions --limit 5 --format raw --no-progress | jq '.'

# Export a session for review
px session <session-id> --include-annotations --format raw --no-progress | \
  jq '{traces: .session.traces, annotations: .annotations}'

# Save a session and ask your AI assistant to analyze it
px session <session-id> --file ./debug-session.json
```
