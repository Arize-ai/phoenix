## Credential Redaction

**Available in arize-phoenix 14.16.0+**

Phoenix now redacts stored secrets and LLM provider credentials in all GraphQL responses. Instead of returning plaintext API keys, the server returns a redacted token that shows a short preview — for example, `sk-•••••1234` — so you can confirm which key is configured without the full value being exposed.

- **Stored secrets** — values set via `PUT /v1/secrets` are returned as redacted strings in queries and mutation payloads
- **Custom provider credentials** — API keys, Azure client secrets, and other credential fields in GraphQL inputs and outputs are all redacted
- **Settings UI** — the Secrets table and replace-secret dialog display redacted previews and use redacted placeholders in credential input fields

This change means stored credentials can no longer be read back in full through the GraphQL API or the UI. Any automation that was reading plaintext credential values from GraphQL responses will need to be updated to treat those fields as opaque tokens.
