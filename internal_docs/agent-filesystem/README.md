# Agent Filesystem Docs

Start here when resuming this work in a new agent session: read `plan.md`, then `tasks.md`, pick the next task in order, check the agents umbrella issue `#11598` for relevant linked issues, and offer to create one or more new GitHub issues if the upcoming task scope is not already tracked.

- `plan.md`: finalized product and architecture plan for the agent filesystem + bash work.
- `tasks.md`: implementation task breakdown with dependencies and suggested execution order.
- `t0.1-tool-contract.md`: v1 frontend contract for client-executed tools in agent chat.
- `t0.2-module-layout.md`: chosen frontend file layout for the bash runtime and context adapters.

Use `plan.md` for the target state and design decisions.
Use `tasks.md` for sequencing the implementation work.
