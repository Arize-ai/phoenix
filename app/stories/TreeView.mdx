# Tree View

The starting primitive for a unified, keyboard-accessible tree used across the
span, trace, and session views. Just bare divs with raw text inside a border —
no indentation, padding, or theming yet. Those arrive as we iterate.

<Canvas of={TreeViewStories.PrimitiveTree} />

## Mouse

- **Click a row** — selects it.
- **Click the selected row again** — opens / closes it (toggles its children).
- **Click the chevron** — opens / closes the row without changing selection.
- Hovering a row highlights it.

## Keyboard

The tree is a single tab stop (roving tabindex): **Tab** moves onto it, **Tab**
again moves past it.

- **↑ / ↓** — move between rows
- **← / →** — collapse / expand a row with children
- **Home / End** — first / last row
- **Enter / Space** — select the focused row (re-press to open it)

Rows show three states: hovered (mouse), focused (keyboard outline), and
selected (filled).

## Nested

The tree nests to arbitrary depth. Children are rendered recursively, and
each row indents by its depth (`level` from react-aria). Only rows that have
children show a chevron; leaves show an empty filler block in the same spot so
labels and row edges stay aligned. The same keyboard and selection model
applies at every level.

tree with its own selection and expansion. A single **span** stack (root → mid →
end); a **trace** wrapping that stack; a **session** of turns; and a **session**
whose children are traces.
