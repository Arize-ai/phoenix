================================================================================
  G. Evaluation Metrics Documentation
================================================================================

  Exact Match (EM)
    Definition: Compares predicted tool-use trajectories against ground truth. Checks whether the selected tool names align precisely -- the complete set (and optionally order) must match.
    Our use:    Core metric for tool selection accuracy. For parallel tasks, order-agnostic set comparison. For sequential tasks, order matters.

  Inclusion
    Definition: Measures the proportion of ground-truth tools present in the predicted trajectory, even if ordering or extra tools differ. Reveals partial task completion.
    Our use:    Useful as a softer metric when agents pick extra tools. Maps to recall of ground-truth tool set.

  Tool Usage (Argument Correctness)
    Definition: Checks if the predicted tool parameters match the ground truth. Validates schema constraints, formats, and value accuracy for tool inputs.
    Our use:    Critical for parameter correctness evaluation. Each tool call's arguments must match expected values. Maps directly to our argument accuracy metric.

  Trajectory Satisfaction (Traj-Satisfy)
    Definition: LLM-judge metric determining to what extent a predicted trajectory can solve the user query (0-10 scale). Used when gold traces are unavailable. Reportedly tracks EM closely.
    Our use:    Useful as LLM-as-judge fallback for open-ended tasks. We can use this approach for tasks where multiple valid trajectories exist.

  Accuracy (Solution Accuracy)
    Definition: Measures if the predicted final answer matches the ground truth answer by prompting an LLM judge. Assesses end-to-end task completion beyond just tool selection.
    Our use:    End-to-end metric. We compare the agent's final response against the ground truth final_answer field.

  Retrieval Rate
    Definition: For retrieval-based methods, measures what proportion of ground truth tools are retrieved from candidate tool sets.
    Our use:    Relevant if we implement tool retrieval/filtering as a preprocessing step before tool selection.

--- Additional sequential-specific metrics ---
  Dependency Satisfaction:
    Definition: Checks whether sequential tool dependencies are respected.
                Output of tool N must be available/referenced by tool N+1.
    Our use:    For sequential tasks, verify the agent respects the
                dependency chain. A tool that needs prior output should
                not be called before its dependency completes.

  Order Satisfaction:
    Definition: Checks if tools are invoked in the correct order for
                sequential tasks.
    Our use:    For sequential tasks, the order of tool calls must match
                the ground truth sequence.
